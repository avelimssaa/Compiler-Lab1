Разработал Зенцов Вадим, АВТ-313

## Постановка задачи

Разработать модуль поиска подстрок с использованием регулярных выражений, 
интегрировать его в существующее приложение (текстовый редактор) и обеспечить наглядный вывод результатов.

## Задание 1.

### Описание задачи.

Построить РВ, описывающее китайские почтовые индексы. 

Китайские почтовые индексы состоят из 6 цифр, первая не может быть 0.

### Регулярное выражение с пояснением каждого обозначения.

(?=([1-9]\d{5}))

?= - означает, что курсор не должен двигаться за пределы найденной группы.

[1-9] - первая цифра, не включая 0.

\d - метасимвол цифры от 0 до 9.

{5} - квантификатор количества повторений, в данном случае 5 цифр.

### Примеры строк, которые должны находиться.

100000  

200000

518000

310000

### Примеры строк, которые не должны находиться.

000000 - начинается с нуля.

12345 - слишком маленькое число.

12ab56 - в середине строки находятся буквы.

## Задание 2.

### Описание задачи.

Построить РВ, описывающее номера карт, принадлежащих платежной системе UnionPay.

Номера данных карт должны начинаться с 62, далее может быть любая цифра. Допустиммый размер номера карт - от 16 до 19.

### Регулярное выражение с пояснением каждого обозначения.

(?=(62\d{14,17}))

?= - означает, что курсор не должен двигаться за пределы найденной группы.

62 - означает, что данная подстрока должна начинаться с 62.

\d - метасимвол цифры от 0 до 9.

{14,17} - квантификатор числа повторений, в данном случае количество цифр может быть от 14 до 17.

### Примеры строк, которые должны находиться.

6292345678901234

6212345678901234567

### Примеры строк, которые не должны находиться.

621234567890123 - слишком мало символов.

6212 3456 7890 1234 - есть пробелы внутри строки.

6222 3456 7890 1234 5 - есть пробелы внутри строки.

6212-3456-7890-1234 - есть неправильные символы внутри строки.

6232-3456-7890-1234-5 - есть неправильные символы внутри строки.

4111111111111111 - начинается не с 62.

62abcdefghijklmn - включает сторонние символы.
